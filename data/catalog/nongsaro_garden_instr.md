# 농사로 OpenAPI `gardenDtl` 코드값 매핑 정리

출처: 첨부 HWP 파일 `OpenAPI 활용 매뉴얼 - 실내정원용 식물.hwp`  
범위: 농사로 OpenAPI **실내정원용 식물 서비스(garden)** 중 `gardenDtl` 상세 응답 코드 필드

---

## 1. 결론

첨부 문서 기준으로, 아래 6개 필드에 대응하는 별도 코드 목록 API operation 이름은 확인되지 않습니다.

- `fmlNm`
- `grwhTpCode`
- `grwtveCode`
- `hdCode`
- `managelevelCode`
- `managedemanddoCode`

즉, 문서에는 `fmlList`, `grwhTpList`, `grwtveList`, `hdList`, `managelevelList`, `managedemanddoList` 같은 operation이 명시되어 있지 않습니다.

다만 문서 하단의 **“※상세코드 정보”** 표에 코드-코드명 매핑이 제공되어 있으므로, 이 표를 기반으로 별도 매핑 테이블을 만들어 사용할 수 있습니다.

## 5. 구현 시 권장 방식

`gardenDtl` 응답에는 코드 필드와 함께 코드명 필드도 함께 내려오는 구조입니다.

| 코드 필드 | 한글명 필드 |
|---|---|
| `fmlNm` | `fmlCodeNm` |
| `grwhTpCode` | `grwhTpCodeNm` |
| `grwtveCode` | `grwtveCodeNm` |
| `hdCode` | `hdCodeNm` |
| `managelevelCode` | `managelevelCodeNm` |
| `managedemanddoCode` | `managedemanddoCodeNm` |

따라서 실제 구현에서는 다음 방식이 안전합니다.

1. `gardenDtl` 응답의 `~CodeNm` 필드를 우선 사용한다.
2. 응답에서 코드명 필드가 비어 있거나 정규화가 필요할 경우, 문서의 상세코드표를 별도 매핑 테이블로 만들어 사용한다.
3. 문서에 없는 코드값은 임의 추정하지 않고 `확인 불가` 또는 `unknown`으로 처리한다.

## 8. 최종 판단

문서 기준 판단은 다음과 같습니다.

- 공식 코드 목록 API operation으로 제공되는 것은 질문자가 확인한 10개가 전부로 보입니다.
- `fmlNm`, `grwhTpCode`, `grwtveCode`, `hdCode`, `managelevelCode`, `managedemanddoCode`는 별도 목록 API명이 문서에 없습니다.
- 대신 문서의 **상세코드 정보** 표를 기준으로 코드-코드명 매핑이 가능합니다.
- 문서에서 확인되지 않는 operation 이름은 추측하지 않는 것이 좋습니다.
